# 收藏工作台项目同步

这个文件夹用来放收藏工作台的“项目同步文件”，方便手动或自动覆盖更新，再通过 GitHub 在不同电脑之间流转。

## 文件

- `ai-channels.json`：项目同步文件。插件读取时采用覆盖式更新，不是累加导入。

## 你会看到的按钮

- `下载备份 JSON`：导出本地备份。
- `导入备份 JSON`：把本地备份合并进当前工作台。
- `导出分享 HTML`：生成可直接打开、可分享给别人看的静态页面。
- `保存项目同步文件`：导出给仓库用的同步文件。优先弹出保存位置，建议直接选择并覆盖 `channel-sync/ai-channels.json`；如果浏览器不支持保存选择器，会下载到默认目录。
- `从项目同步文件更新`：读取项目里的同步文件并覆盖更新当前工作台。

## 推荐流程

1. 在收藏工作台打开“管理”。
2. 点击“保存项目同步文件”。
3. 在保存位置里选择本文件夹的 `ai-channels.json` 并覆盖；如果文件被下载到默认目录，就手动复制过来覆盖。
4. 提交并推送到 GitHub。
5. 另一台电脑拉取项目后运行构建，或确认 `dist/channel-sync/ai-channels.json` 已更新。
6. 刷新/重载插件。插件会在扫描时自动尝试读取同步文件，也可以手动点“从项目同步文件更新”。

## 注意

- 浏览器扩展不能静默写入项目目录。新版会尽量弹出保存位置让你手动选择目标文件；如果浏览器不支持这个能力，就只能下载后再由你覆盖仓库文件。
- `scripts/postbuild.mjs` 会把本文件夹复制到 `dist/channel-sync/`。如果大家都是加载 `dist/` 使用插件，这一步是必须的。
- 同步按 URL 更新记录，并替换当前工作台数据，不会把旧记录越导越多。
